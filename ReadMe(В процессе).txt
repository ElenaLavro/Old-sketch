Теория Git
Что же такое Git, если говорить коротко? Очень важно понять эту часть материала, потому что если вы поймёте, что такое Git и основы того, как он работает, тогда, возможно, вам будет гораздо проще его использовать. Пока вы изучаете Git, попробуйте забыть всё, что вы знаете о других СКВ, таких как Subversion и Perforce. Это позволит вам избежать определённых проблем при использовании инструмента. Git хранит и использует информацию совсем иначе по сравнению с другими системами, даже несмотря на то, что интерфейс пользователя достаточно похож, и понимание этих различий поможет вам избежать путаницы во время использования.

Теория HTML
HTML - это не язык программирования, это язык разметки , которая говорит вашему браузеру, как отображать содержимое веб-страницы, которую вы просматриваете. Она может быть простой или сложной, в зависимости от желания веб-дизайнера, который ее создает. HTML состоит из серии ( элементов ) , которые используются для размещения, или "заворачивания" в них различных частей контента, чтобы они отображались действовали определенным образом. Эти элементы с помощью начального и конечного ( тегов ) могут сделать слово или изображение ссылкой на любую другую страницу, могут отображать текст курсивом, увеличить или уменьшить шрифт и тому подобное.

Моё мнение:
HTML является первым языком web-программирования. На его основе построены многие команды на php, jsи т.д. Таким образом можно сказать, что он вечен. При написании сайта на чистом HTML  на странице нет ничего лишнего, она быстро загружается, хорошо доступна для поисковых роботов, легче оптимизируется. В HTML можно продумать свою Иерархию страниц, выделить главные и провести добротную внутреннюю оптимизацию сайта.
А также, HTML достаточно легкий в изучении.

Я думаю что Git - это прекрасная платформа для всех веб-разработчиков. Во-первых, он имеет распределенную архитекутуру разработки, совместимость с существующие протоколами, а также способность обрабатывать крупномасштабные проекты и множество вариантов. Git имеет мало негативов, один из них - низкоуровневый интерфейс(хотя некоторым может и нравится).
